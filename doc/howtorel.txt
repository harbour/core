How to create and tag a new release
===================================
Updated for Git workflow (2026).


Prerequisites
-------------

* Clean working tree on the release commit
* All CI workflows green (linux, windows, macos, vm1, vm2)
* Agreement on harbour-devel about version number and release date


1. Prepare version files
------------------------

Update the Harbour version in these files (search for the current values):

    include/hbver.h
    config/global.mk
    package/harbour.spec
    package/harbour-win.spec.in
    package/harbour-wce.spec.in
    debian/changelog
    package/winuni/RELNOTES.txt

For a release candidate, set HB_VER_STATUS to "rc1" or "rc2".
For the final release, set HB_VER_STATUS to "" (empty string).


2. Update ChangeLog and commit
------------------------------

    hbrun bin/commit
    git commit -am "Release 3.2.0"

Use the standard ChangeLog entry format. The commit hook enforces this.


3. Tag the release
------------------

    git tag -a v3.2.0 -m "Harbour 3.2.0"
    git push origin master
    git push origin v3.2.0

For release candidates use tags like v3.2.0rc1.


4. Build source package
-----------------------

    package/mpkg_src.sh

Produces harbour-<version>.src.tar.gz in the package directory.


5. Build binary packages
------------------------

From the Harbour root directory:

*nix:*

    export HB_BUILD_PKG=yes
    make clean install
    package/mpkg_rpm.sh

*Windows:*

    set HB_BUILD_PKG=yes
    win-make clean install

Or use package\winuni\mpkg_win_uni.bat for the unified installer.
See README.md section "How to Create Distributable Packages".


6. Publish
----------

Upload artefacts to:

    https://github.com/harbour/core/releases
    https://sourceforge.net/projects/harbour-project/files/

Attach source archive, Windows binaries, and release notes summarizing
changes since the previous stable release (derive from ChangeLog.txt).


7. Bump development version
---------------------------

After publishing, on master:

* Increment HB_VER_RELEASE or HB_VER_MINOR in hbver.h / global.mk
* Set HB_VER_STATUS back to "dev"
* Add a ChangeLog entry for the version bump
* Commit and push


8. Update the website
---------------------

In harbour/harbour.github.io:

* _config.yml: stable_version, stable_date
* _data/news.yml: release announcement
* index.html picks up config automatically

See https://github.com/harbour/harbour.github.io
